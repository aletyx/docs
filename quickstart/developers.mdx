This path is for **developers** who integrate, customize, and extend Aletyx in
code. You'll scaffold a Spring Boot project with the Kogito Maven archetype and
run it locally.

## Prerequisites

- Java and Maven installed
- A Maven access token for the Aletyx enterprise builds (if required)

See [environment setup](https://docs.aletyx.ai/getting-started/environment-setup/)
for the full workstation configuration.

## Scaffold and run a service

<Steps>
  <Step title="Generate a new module">
    Generate a new module, either using your prefered tool or the [Kogito Spring Boot archetype](https://mvnrepository.com/artifact/org.kie.kogito/kogito-spring-boot-archetype/10.2.0),
    setting your own Group ID, Artifact ID, and Version:

    ```bash
    mvn archetype:generate \
        -DarchetypeGroupId=org.kie.kogito \
        -DarchetypeArtifactId=kogito-spring-boot-archetype \
        -DarchetypeVersion=10.2.0 \
        -DgroupId=com.example \
        -DartifactId=my-decision-service \
        -Dversion=1.0.0-SNAPSHOT
    ```

    Your favorite Java tooling works too — every major IDE can create a Maven
    project from an archetype (for example, IntelliJ IDEA's *New Project → Maven
    Archetype* or the Eclipse Maven project wizard) using the same coordinates.
  </Step>
  <Step title="Build it">
    `cd` into the new directory and run `mvn clean install`.
  </Step>
  <Step title="Run it">
    Launch with `mvn spring-boot:run`, then open `localhost:8080`.
  </Step>
  <Step title="Customize and extend">
    Modify decision models, add configuration, connect data sources, and
    implement your business logic.
  </Step>
</Steps>

<Check>
  You have a running Aletyx service scaffolded from the archetype. Integrate it
  with your systems via the API or call processes directly from Java.
</Check>
