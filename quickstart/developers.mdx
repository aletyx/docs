This path is for **developers** who integrate, customize, and extend Aletyx in
code. You'll scaffold a project with [start.aletyx.ai](https://start.aletyx.ai)
and run it locally.

## Prerequisites

- Java and Maven installed
- A Maven access token for the Aletyx enterprise builds (if required)

See [environment setup](https://docs.aletyx.ai/getting-started/environment-setup/)
for the full workstation configuration.

## Scaffold and run a service

<Steps>
  <Step title="Open the project starter">
    Go to [start.aletyx.ai](https://start.aletyx.ai).
  </Step>
  <Step title="Configure your project">
    Set the Group ID, Artifact ID, Version, Runtime (Spring Boot or Quarkus),
    target version, and Maven access token.
  </Step>
  <Step title="Select a template">
    Choose a starting point — for example *Intelligent Decision Service*,
    *Decision Service*, *Rule Service*, *Hybrid Decision Service*, *Decision
    Orchestration*, or *Workflow Automation*.
  </Step>
  <Step title="Generate and download">
    Click **Download Project** to get a pre-configured ZIP.
  </Step>
  <Step title="Build it">
    Unzip and run `mvn clean install`.
  </Step>
  <Step title="Run it">
    Launch with `mvn spring-boot:run` (Spring Boot) or `mvn quarkus:dev`
    (Quarkus), then open `localhost:8080`.
  </Step>
  <Step title="Customize and extend">
    Modify decision models, add configuration, connect data sources, and
    implement your business logic.
  </Step>
</Steps>

<Check>
  You have a running Aletyx service scaffolded from a template. Integrate it with
  your systems via the API or call processes directly from Java.
</Check>
